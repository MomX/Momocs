# Coo "super" class

`Coo` class is the 'parent' or 'super' class of
[`Out`](http://momx.github.io/Momocs/reference/Out.md),
[`Opn`](http://momx.github.io/Momocs/reference/Opn.md) and
[`Ldk`](http://momx.github.io/Momocs/reference/Ldk.md) classes.

## Usage

``` r
Coo(...)
```

## Arguments

- ...:

  anything and, anyway, this function will simply returns a message.

## Value

a list of class Coo

## Details

Useful shortcuts are described below. See `browseVignettes("Momocs")`
for a detail of the design behind Momocs' classes.

`Coo` class is the 'parent' class of the following 'child' classes

- [`Out`](http://momx.github.io/Momocs/reference/Out.md) for closed
  **out**lines

- [`Opn`](http://momx.github.io/Momocs/reference/Opn.md) for
  **op**e**n** outlines

- [`Ldk`](http://momx.github.io/Momocs/reference/Ldk.md) for
  configuration of **l**an**d**mar**k**s

Since all 'child classes' of them handle \\(x; y)\\ coordinates among
other generic methods, but also all have their specificity, this
architecture allow to recycle generic methods and to use specific
methods.

In other words, [`Out`](http://momx.github.io/Momocs/reference/Out.md),
[`Opn`](http://momx.github.io/Momocs/reference/Opn.md) and
[`Ldk`](http://momx.github.io/Momocs/reference/Ldk.md) classes are all,
primarily, `Coo` objects on which we define generic *and* specific
methods. See their respective help pages for more help.

`Coo` objects all have the following components:

- `$coo` which is a list of matrices for coordinates

- `$fac` a `data_frame` for covariates (if any). You can provide this
  data_frame directly, as long as it has as many rows as there are
  matrices in `$coo` (see examples), or use an helper function such as
  [lf_structure](http://momx.github.io/Momocs/reference/lf_structure.md).

You can access all the methods available for `Coo` objects with
`methods(class=Coo)`.
